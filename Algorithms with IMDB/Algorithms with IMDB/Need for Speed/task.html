Description

Have you noticed how much time it takes for a linear search to fetch anything? The number of iterations with this algorithm is directly proportional to the number of elements in the data structure. Suppose our dataset contains one million elements. Does it necessarily mean one million iterations? Not always, but in the worst case, yes. Especially, if the target element tends to the end of the dataset. The time complexity for this algorithm is \(O(n)\).

To speed up the process and reduce the number of iterations, it is a good idea to implement a more efficient algorithm called binary search (also referred to as logarithmic search). Binary search works with a sorted collection of elements only â€” that's why you've sorted the array in the previous stage. The time complexity of the binary search algorithm is \(O(log~n)\).

In this stage, you need to implement the binary search algorithm and print the movie titles with a rating of 6.

Tip: Factoid time! If you put down all names in the world to search for a specific name, a binary search can find what you're looking for in 35 iterations.

Objectives

To complete this stage, your program should do the following:

  1. Take a sorted data collection, add the following variables: low = 0 and high = n - 1, where n is the number of elements in the collection, to store the index of the first and the last element;
  2. Check the rating of the middle element in the collection. If it's equal to 6, print the movie title along with others;
  3. If the rating is less than 6, set low = middle + 1. It should be done with every iteration;
  4. If the rating is greater than 6, set high = middle - 1. Repeat the procedure, starting from Step 2.
