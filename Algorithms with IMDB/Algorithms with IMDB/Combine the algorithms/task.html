Description

By combining bubble sort and binary search algorithms, you could speed up a search to a great extent, but it still takes a lot to sort the input data with bubble sort. In this stage, you will further improve total time complexity by implementing merge sort. After sorting, implement binary search like in the previous stage.

The worst-case time complexity for merge sort is \(O(n~log~n)\), which is better than \(O(n^2)\) of bubble sort. Make notes of the total time in this case.

Objectives

Your program should do the following:

  1. Take the unsorted data and divide the collection into two parts;
  2. Sort the left part with the merge sort algorithm. Divide the sub-parts further into equal parts until a single element remains and merge them;
  3. Sort the right part using the merge sort algorithm. Divide the sub-parts further into equal parts until a single element remains and merge them;
  4. Merge the resulting left and right collections by comparing the elements (ratings) and form a single sorted collection as a result;
  5. Once the collection is sorted, apply binary search and print the movie titles with a rating of 6.
Example

Example 1: output of your program

Hercules - 6.0
The Fast and the Furious: Tokyo Drift - 6.0
Finding You - 6.0
